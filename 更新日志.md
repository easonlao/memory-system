## v4.1.2（2026-05-07）

### 修复：Skill 模板自包含

- Skill 步骤1：新增平台检测代码块（`if USERPROFILE` 判断）+ 路径映射速查表
- Skill 步骤2：三端路径配置全部升级为双列路径表（Unix 路径 / Windows 路径）
- 每个路径配置块前新增 Windows 路径转换提醒
- 4处「本文件系统常量章节」外部引用 → 全部改为自包含引用

**影响文件**：
| 文件 | 改动 |
|------|------|
| WorkBuddy版执行文件 | 4处外部引用 → 内嵌速查表 |
| Qclaw版执行文件 | 步骤1+步骤2全面升级（与WorkBuddy版对齐） |

## v4.1.1（2026-05-06）

### 新增：I0 初始化执行铁律（断点恢复 + 防编造）

**背景**：新用户在执行初始化过程中遇到对话中断（context用完、网络断开等），重新开始时AI凭记忆跳步骤或编造内容，导致初始化结果不完整或不正确。

**新增内容**（行为铁律章节，原 I1 之前插入）：
- **读本文件才能操作**：执行文件不在当前上下文时，必须先读取再操作，不得凭记忆复述
- **断点检测流程（9步文件检测）**：按顺序检查已生成的文件，定位中断位置
- **严禁凭记忆操作**（5条禁止对照表）：禁止跳步骤、禁止编造路径、禁止凭印象判断文件状态
- **用户质疑时处理**：必须回读文件验证，不得自证

**影响文件**：三端执行文件均同步新增 I0 铁律。

#### 2. 软链接同步体系（从"建议"升级为架构级机制）

**v4.0 状态**：皮叔执行版中软链接只是"建议"（一行提示），没有强制写入初始化步骤。

**v4.1 变更**：软链接成为初始化流程的**必选步骤**，写入第6-8步：

| 同步对象 | 方式 | 链接方向 | 频率 |
|---------|------|---------|------|
| **Skill 文件** | 软链接（`ln -s`） | `[记忆共享中心]/技能配置/*/SKILL.md` → `~/.workbuddy/skills/*/SKILL.md` | 实时（改主版本即生效） |
| **规则文件** | 软链接 + launchd 双保险 | `[记忆共享中心]/记忆规则/*.md` → `~/.workbuddy/rules/*.mdc` | 软链接实时 + launchd 每小时兜底 |
| **路由表** | 写入 | 初始化时根据 G3b 模板生成 `~/.workbuddy/rules/记忆路由表.mdc` | 一次性 |

**三端软链接映射**：

| 平台 | Skill 链接目录 | 规则链接目录 |
|------|--------------|------------|
| WorkBuddy | `~/.workbuddy/skills/` | `~/.workbuddy/rules/` |
| Qclaw | `~/.openclaw/skills/` | `~/.openclaw/rules/` |
| Trae | `~/.trae/skills/` | Trae 用脚本聚合，不走软链接 |

**核心原则**：
- 所有 Skill 的**主版本**统一存放在 `~/[记忆共享中心]/技能配置/`
- 各平台目录下的 Skill 是软链接，**改主版本 → 三端同时更新**
- launchd 定时脚本作为软链接的兜底，防止链接断裂时规则不同步

#### 4. 平台路径映射规则（2026-04-28）

**背景**：执行文件中的路径模板使用 Unix 风格，Windows 用户（如 Qclaw）需要手动转换。

**新增内容**（系统常量章节）：
- 平台检测方式（环境变量判断）
- `~/` 跨平台映射规则（`$HOME/` vs `%USERPROFILE%\`）
- 路径分隔符规则
- 路径转换伪代码
- 各平台路径对照表

#### 5. WorkBuddy 注入路由器（v4.0.1 遗留，v4.1 纳入）

**背景**：Qclaw 依赖心跳机制做路由，WorkBuddy 没有心跳但 rules 全量注入。

**方案**：
- 创建 `记忆路由表.mdc`，利用 rules 每次会话 100% 注入特性
- 路由触发降级为信号词匹配（降低执行门槛）
- 加否定词排除（避免误触发）

#### 6. Qclaw 版特化（v4.2 标记）

- Qclaw 版文件自标 v4.2（2026-04-28）
- 路径适配为 `~/.openclaw/`
- 规则目录映射为 `~/.openclaw/rules/*.md`
- L4 情境层为 `00_动态状态快照.md`（非 MEMORY.md）
- L5 拆分为长期记忆（`LONG_TERM_MEMORY.md`）+ 潜意识（`SHADOW.md`）

#### 7. Trae 版特化

- 路径适配为 `~/.trae/`
