# 02 — 入口与加载机制

> 定义记忆系统如何被接入、AI 每次对话的加载顺序。

---

## 系统接入架构

```
AGENTS.md / CLAUDE.md / .cursorrules / GEMINI.md
  │ 平台入口文件，写入一行引用
  ▼
INDEX.md（五层记忆系统入口）
  │ AI 首次对话时读取，了解系统结构和路由
  ▼
路由表（rules/记忆路由表.md）
  │ 每次对话生效，根据用户消息信号词决定读取什么
  ├── 信号词匹配 → 读取对应文件
  └── 无匹配 → 跳过
```

### 平台入口文件配置

在每个平台的入口文件（如 AGENTS.md）中追加一行：

```
> 记忆系统入口：$MEMORY_ROOT/五层记忆系统/INDEX.md
```

确保新 agent 启动时能自动发现记忆系统。

---

## 每次对话加载顺序（G2）

> AI 按以下顺序确认记忆已加载。前两阶段由平台注入机制负责，无需手动操作。

```
# 第一阶段：身份与铁律（平台注入 + 自动加载）
1. GL1 铁律     — AI 不可违的行为底线（$MEMORY_ROOT/成长箱/01-铁律.md）

# 第二阶段：情境（平台注入或 AI 按需读取）
5. MEMORY.md    — 当前目标/卡点/TTL 状态（L4）

# 第三阶段：按需加载（AI 根据路由表信号词判断）
6. L2/L3 规则   — 详细认知模式和行为规律（讨论决策/排计划时）
7. 当日+昨日日记 — 回顾/规划/写日记时
8. 相关 SKILL   — 执行特定任务时
9. GL2 模式候选  — 当需要参考合作规律时
```

### 不用手动加载的内容

| 文件 | 原因 |
|------|------|
| L1 CORE.md | 已编译为 GL1 铁律，无需读原文 |
| L2/L3 详细规则 | 日常对话不需要，路由表触发时才读 |
| L4 项目日志 | 按项目隔离，讨论对应项目时才读 |

## 什么时候需要手动加载

| 场景 | 读取内容 |
|------|---------|
| 初始化/重装系统 | 读 `07-问卷.md` 引导问卷 |
| 做决策分析 | 读 L2（认知模式）+ L1（价值观） |
| 写周报 | 读本周所有日志 + L4 |
| 月度蒸馏 | 读本月所有日志 + L3/L2/L1 + 成长箱 |
| 写日记 | 读昨日日记 |
| 读书笔记 | 读 `08-技能模板/读书助手.md` |

## 路由机制

路由表位于 `$MEMORY_ROOT/rules/记忆路由表.md`，由以下内容组成：

1. **信号词 → 文件映射** — 见 `INDEX.md` 路由表
2. **工具类路由** — 信号词触发 Skill（日记→记忆助理、读书笔记→读书助手）
3. **兜底规则** — 无匹配时正常回复，末尾标记 `[🧠 路由: 无匹配]`
4. **否定词排除** — 信号词前有"不/没/别"时不触发

每次回复末尾追加一行路由状态标记，确保路由可观测。
