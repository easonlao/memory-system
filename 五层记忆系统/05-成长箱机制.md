# 05 — 成长箱机制

> 定义 AI 侧的规则系统：GL1 铁律、GL2 模式候选、晋升流程。
> 成长箱是 AI"长记性"的地方——不是关于你是什么人，而是关于怎么更好地跟你合作。

---

## 成长箱结构

```
$MEMORY_ROOT/成长箱/
├── 01-铁律.md           ← GL1：所有 AI 必须遵守的规则（每次对话必读）
└── 02-模式候选.md       ← GL2：观察中的合作规律（按需读取）
```

---

## GL1 铁律

**定义**：AI 不可违的行为底线。从你的 L1 编译 + 跨平台通用规则。

**特性**：
- 每次对话必须加载
- 你确认后永久锁定，不衰减
- 跨平台共享（所有 AI 通用）

**来源**：

| 来源 | 说明 | 初始置信度 |
|------|------|-----------|
| 你的 L1 编译 | 从你的价值观/红线直接翻译 | 0.9 |
| GL2 跨项目晋升 | 同一规则在 2+ 项目中被证实有效 | 0.7，你确认后 0.9 |

**示例**：
```
- 修改文件前必须输出修改清单等用户确认
- 没数据时不编造，如实说"没有"
- 不确定用户意图时先问清楚
```

---

## GL2 模式候选

**定义**：观察中的合作规律——还没到铁律程度，但值得记录的"什么方式有效"。

**特性**：
- 按需读取，不占上下文
- 置信度管理（0.3-0.7）
- 支持不同粒度

**分层**：

| 分层 | 范围 | 存储 | 晋升条件 |
|------|------|------|---------|
| GL2a 场景规则 | 跨项目通用（"对比任务用表格"） | `$MEMORY_ROOT/成长箱/02-模式候选.md` | 置信度 ≥ 0.7 → 可升 GL1 |
| GL2b 项目规则 | 特定项目有效 | `{项目}/.workbuddy/.rules/` | 同规则在 2+ 项目出现 → 升 GL2a |
| GL2c 临时约定 | 当前对话约定（"这次用中英对照"） | 不持久 | 对话结束即过期 |

**示例**：
```
(置信度 0.5) 当涉及方案对比时，优先用表格呈现
(置信度 0.7) 用户情绪低落时，先共情再给建议
(置信度 0.3) 项目 A 中用户喜欢先看结论再看分析
```

## 晋升流程

```
GL2b 项目规则（某个项目中发现有效）
  │ 同规则在 2+ 项目中出现
  ▼
GL2a 场景规则（跨项目通用，置信度 0.5）
  │ 累计 5+ 次有效 + 你明确肯定
  ▼
GL2a 置信度 0.7（稳定模式）
  │ 跨项目 + 持续有效
  ▼
GL1 提案（AI 向你确认）
  │ 你确认
  ▼
GL1 铁律（永久生效）
```

### 晋升评估频率

| 评估时机 | 评估内容 | 动作 |
|---------|---------|------|
| 每次对话结束 | 本次对话有新的合作规律吗？ | 写入 GL2c 临时笔记 |
| 周报时 | 检查 GL2 候选的置信度 | 更新置信度 |
| 月度蒸馏时 | 批量评估晋升 | 执行晋升/归档 |

### 晋升规则

1. **GL2b → GL2a**：同一规则在 2+ 项目出现 → 自动晋升。不需要你确认。
2. **GL2a → GL1**：置信度 ≥ 0.7 + AI 提案 → 你确认后才晋升。需要你确认。
3. **GL2 置信度衰减**：3 个月未触发 → 每月降 0.1。降到 0.3 以下归档。

## 错误晋升（原成长箱）

当 AI 在同类型任务上犯错时，错误记录本身也遵循成长箱逻辑：

```
AI 犯错
  │
记录到 AI 操作日志
  │ 同类错误多次
  ▼
GL2 候选："这类错误需要规则防范"
  │ 你确认
  ▼
GL1 铁律
```

但与用户规律不同的是：**AI 错误晋升的规则只在当前平台生效**，不跨平台共享。
