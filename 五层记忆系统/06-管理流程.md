# 06 — 管理流程

> 定义记忆系统的三个核心工作流。分为主动（你说"做XX"）和自动（AI 自己沉淀）两类。

---

## 工作流总览

| # | 工作流 | 主动/自动 | 触发 | 产出 |
|---|--------|----------|------|------|
| 1 | 工作区记忆维护 | 自动 | 每次对话 | 更新 L4 MEMORY.md |
| 2 | 月度校准 | 主动 | 你说"月度校准" | 更新 L3/L2/L1 |
| 3 | 成长箱沉淀 | 自动 | 每次对话 + 月度校准 | 更新 GL2/GL1 |

---

## 工作流 1：工作区记忆维护（自动）

> AI 每次对话自动维护 L4，详见「工作区记忆维护」Skill。

### 触发时机

由 README.md 自动化路由判断触发（完成结果 / L4 明显变化）。

### 执行

调用「工作区记忆维护」Skill，按模板写入 L4 并同步到项目总览。

### 每次对话开始时

```
1. 检查 $WORKSPACE/.memory/MEMORY.md 是否存在
   ├── 存在 → 读取，了解当前工作区状态
   └── 不存在 → 稍后由 Skill 创建
```

**注意**：L4 是 AI 的工作区笔记，不是用户日记，不是用户画像。

## 工作流 2：月度校准（主动）

> 你说"月度校准"时执行。从日记中提炼用户画像，逐层升级。
> 这是最重要的流程，AI 应逐步骤执行并输出中间结果。

### 输入源

用户日记（工作区 daily records），不是 L4。

### 步骤 ①：日记 → L3 提炼

> 从本月日记中提取行为规律

```
输入：本月所有日记
处理：扫描出现 3+ 次以上的行为模式
输出：L3 候选列表（含置信度和证据来源）
写入：$MEMORY_ROOT/记忆规则/用户基本规则.md（行为段）

例：
"本月 3 个项目都在截止日前 2 天集中冲刺"
→ L3 候选 "截止日前冲刺" 置信度 0.5
```

### 步骤 ②：L3 → L2 归纳

> L3 中置信度 ≥ 0.7 的模式升级为 L2 认知模式

```
输入：L3 当前列表（关注置信度 ≥ 0.7 的条目）
处理：
  - 检查该模式是否已存在 L2
  - 存在 → 合并证据，提升置信度
  - 不存在 → 创建 L2 新条目
输出：更新的 L2 列表
写入：$MEMORY_ROOT/记忆规则/用户基本规则.md（认知段）
```

### 步骤 ③：L2 → L1 提案

> L2 中置信度 ≥ 0.9 持续 3 个月的作为 L1 候选

```
输入：L2 当前列表（关注置信度 ≥ 0.9 的条目）
处理：
  - 该模式已稳定持续 3 个月以上？
  - 跨场景一致？（工作/生活/多个项目）
  - 反例极少？
输出：L1 候选提案
动作：向你输出提案，等你确认后才写入 L1

例：
"过去 6 个月，你在所有项目中的决策模式都是
'先充分收集信息再行动'，且多次明确表示
'不喜欢在不清楚全貌时做决定'。
建议纳入 L1：'信息充分是决策的前提'"
```

### 步骤 ④：L1 ↔ L3 一致性审计

> 对比 L1 说"你重视什么"和日记显示"你实际做了什么"

```
处理：
  - 逐条比对 L1 值与本月日记数据
  - 计算差异
  - 标记持续矛盾的条目

输出格式：
📊 L1↔L3 一致性审计
━━━━━━━━━━━━━━━━
 ✅ L1「结果重于过程」↔ 本月完成率 85% → 一致
 ⚠️ L1「健康优先」↔ 本月平均睡眠 5.2h → 差异显著
 ❌ L1「不将就」↔ 本月勉强接受了 2 次合作 → 建议讨论
```

### 月度校准输出汇总

```
📋 本月校准结果
━━━━━━━━━━━━━━━━
L3 提炼：3 条新行为规律（置信度 0.5-0.7）
L2 归纳：2 条升级为认知模式
L1 提案：1 条待确认（见上方）
L1↔L3 审计：1 处显著差异建议关注
```

## 工作流 3：成长箱沉淀（自动）

> AI 自动维护，从对话和工作中沉淀合作规律。

### 子工作流 3a：对话结束自检（每次对话）

```
1. 本次对话中，有没有发现新的合作规律？
   → 有？置信度多少？→ 写入 GL2c（临时笔记）
   → 没有？跳过

2. 本次对话中，用户有没有明确肯定或否定某种工作方式？
   → 肯定某方式 → GL2c 对应条目的置信度 +0.1
   → 否定某方式 → 对应条目置信度 -0.1

3. 本次对话中，有没有触及已有的 GL1 规则？
   → 触及 → 确认规则是否仍然有效
```

### 子工作流 3b：成长箱晋升（月度校准顺便执行）

> 月度校准跑完用户侧提炼后，顺便跑成长箱晋升。

```
1. 检查 GL2 置信度
   - 标记跨项目出现的规则
   - 核对置信度数据

2. 晋升评估
   - GL2 置信度 ≥ 0.7 且跨 2+ 项目 → 向你提案晋升 GL1
   - 你确认后写入 GL1

3. 从 L1/L2 编译 GL1
   - L1/L2 本次有更新？→ 同步编译到 GL1
   - L1/L2 无变化？→ 跳过

4. 归档
   - GL2 置信度降到 0.3 以下 → 归档
```

## 置信度规则参考

详细置信度规则见 `03-置信度体系.md`。

| 层级 | 0.3 | 0.5 | 0.7 | 0.9 |
|------|-----|-----|-----|-----|
| L3 | 同一项目 3-5 天 | 同一项目 2 周 | 跨 2+ 项目 | 跨项目 3 个月 |
| L2 | 你 1-2 次表达 | 3-5 次 + L3 佐证 | 5+ 次 + 跨项目佐证 | 6 个月 + 无反例 |
| L1 | — | — | AI 提案阶段 | 你已确认 |
| GL2 | 1 次有效 | 2-3 次同场景 | 5+ 次跨场景 | 跨项目 + 你肯定 |
| GL1 | — | — | 从 GL2 晋升 | 你已确认 |
