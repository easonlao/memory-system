# 五层记忆系统 v5.0 — AI 执行入口

`$MEMORY_ROOT` = `E:\SynologyDrive\Memory\源文件`

---

## 一、每次对话开始时的固定动作

```
1. 执行 `pwd` → 得到 $WORKSPACE

2. 检查 $WORKSPACE/.memory/MEMORY.md 是否存在
   ├── 存在   → 读取，获取用户当前目标/卡点
   └── 不存在 → 创建该目录结构（mkdir -p）

3. 检查 $WORKSPACE/.memory/daily/ 最近的工作区日记
   ├── 有今日/昨日日记 → 读取，了解对话脉络和衔接点
   └── 没有 → 跳过

4. 读取 $MEMORY_ROOT/成长箱/01-铁律.md（GL1）
   → 你的行为底线

5. 读取 $MEMORY_ROOT/成长箱/02-模式候选.md（GL2）
   → 用户的交互偏好和协作规律

6. 路由表就绪
   → 接下来每次用户消息，扫描路由表信号词
```

---

## 二、路由表（每次回复前扫描）

> 分两类：信号词（用户说了什么）和自动化（对话中判断条件）。
> 匹配多条则全部执行。信号词前有"不/没/别"时不触发。

### 信号词路由（被动响应）

| 信号词 | 动作 |
|--------|------|
| 写日记 / 今天的日记 | 调用「每日伙伴」Skill |
| 写周报 / 周复盘 | 调用「每日伙伴」Skill |
| 月度校准 / 月度蒸馏 | 读取 `06-管理流程.md` 工作流2（月度校准）→ 逐步骤执行 |
| 成长箱 / 铁律 / 晋升 | 读取 `05-成长箱机制.md` |
| 检查记忆 / 记忆状态 | 调用「唤醒记忆系统」Skill |
| 计划 / 今天做啥 / 明天 | 读取 `01-架构定义.md` 的 L3 行为规律节 → 参考其节奏排计划 |
| 决策 / 要不要 / 选A还是B | 读取 `01-架构定义.md` 的 L1+L2 节 → 参考价值观+认知模式 |
| 更新我的 XX 层 | 读取 `07-问卷.md` 对应章节 → 引导用户重新回答 |
| 项目总览 / 所有项目 / 工作区总览 | 读取 `$MEMORY_ROOT/项目总览.md` → 汇总当前所有工作区 |
| 记录工作区 / 记工作日志 / 今天的工作 | 调用「工作区记忆维护」Skill 工作流 A → 写工作区日记 |

### 自动化路由（主动判断）

> 以下触发条件有明确定义，AI 按定义判断是否触发，不凭感觉。

| 触发条件 | 定义 | 动作 |
|---------|------|------|
| [自动] 完成一个确定的结果 | 满足任一：①用户说"搞定/可以了/就这样"；②AI 交付完整产物（代码/方案/分析）；③用户确认采用方案；④PR/issue 关闭 | 调用「工作区记忆维护」Skill 工作流 A → 写工作区日记 |
| [自动] 发现同一类错误反复出现 | 用户在同一个问题上纠正 AI ≥ 2 次 / 同一操作连续报错 ≥ 2 次 / 用户重复提同样的修改要求 | 记录到成长箱 GL2c（含错误描述 + 出现次数）|
| [自动] L4 状态明显变化 | 上次更新 L4 后，工作区出现新项目 / 目标完成 / 卡点变化 / 或上次更新距今已过多次对话 | 主动询问："工作区状态变了，要更新工作区记忆吗？" |

每次回复末尾追加一行：`[🧠 路由: {信号词匹配项} | {自动化触发项}]`
没有任何匹配/触发则追加：`[🧠 路由: 无]`
