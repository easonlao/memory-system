# 01 — 架构定义

> 定义Agent-Memory的核心结构：每一层的含义、存储位置、数据流向。

---

## 层级架构

| 层级 | 名称 | 内容 | 变化速度 | 存储位置 |
|------|------|------|---------|---------|
| L1 | 元信息层 | 价值观、底线、核心驱动力、金句 | 年/永久 | `$MEMORY_DATA/USER/core.md` |
| L2 | 认知模式 | 决策习惯、思维方式、交互偏好 | 季度 | `$MEMORY_DATA/USER/cognitive.md` |
| L3 | 行为规律 | 能量周期、工作节奏、习惯 | 月度 | `$MEMORY_DATA/USER/behavior.md` |
| L4 | 情境状态 | 当前目标、卡点、项目进度 | 天/周 | `$WORKSPACE/.memory/MEMORY.md` |

### L1 元信息层

**定义**：你最底层的价值观、底线、核心驱动力。你"是什么样的人"的本质。

**特点**：
- 变化极慢，写一次管很久
- 必须有你确认才能写入或修改
- 不衰减，永久有效直到你主动改变

**来源**：
- 问卷 Q1-Q7
- L2 中持续 6 个月以上稳定的模式（AI 提案 + 你确认）

**文件**：`$MEMORY_DATA/USER/core.md`

### L2 认知模式层

**定义**：你习惯怎么思考、怎么做决策、怎么接收信息。

**特点**：
- 变化较慢（季度级）
- 置信度管理，自动写入但需佐证
- 6 个月未观察到开始衰减

**来源**：
- 问卷 Q8-Q17
- 从 L3 行为规律归纳（AI 自动 + 置信度评分）

**文件**：`$MEMORY_DATA/USER/cognitive.md`

### L3 行为规律层

**定义**：从项目级数据中提取的跨项目共性行为模式。

**特点**：
- 变化中等（月级）
- 从 L4 原始数据自动提取，不需要你确认
- 数据不出现后每周降置信度

**来源**：
- 多个项目的 L4 日记/进度数据泛化
- 对话中的行为观察

**文件**：`$MEMORY_DATA/USER/behavior.md`

### L4 情境状态层

**定义**：你现在在做什么，当前项目的状态、目标和进度。

**特点**：
- 变化最快（天/周级）
- 按项目隔离，不进入公共区
- AI 自动记录，不需要提炼

**来源**：
- 日常对话
- AI 自动记录日志/进度

**文件**：`$WORKSPACE/.memory/MEMORY.md`

## 数据流向

### 提炼方向（从具体到本质）

```
L4 项目数据（日志/进度/任务）
  │ AI 在月度校准中提取跨项目共性
  ▼
L3 行为规律（"截止日前你会冲刺"）
  │ 置信度 ≥ 0.7 → 考虑晋升 L2
  ▼
L2 认知模式（"你习惯先分析再行动"）
  │ 置信度 ≥ 0.9 持续 3 个月 → AI 提案
  ▼
L1 元信息（"结果比过程重要"）
  │ 你确认后锁定
```

### 预测方向（从本质到具体）

```
L1（"结果比过程重要"）
  │
L2（"截止日前会熬夜赶工"）
  │
L3（"最后 48 小时效率最高"）
  │
L4（"项目 A 还剩 3 天截止"）
```

## AI 侧结构

### 组件定义

| 组件 | 内容 | 存储位置 | 加载方式 |
|------|------|---------|---------|
| GL1 铁律 | AI 不可违的行为底线，从 L1 编译 + GL2 晋升 | `$MEMORY_DATA/AGENTS/GL1-rules.md` | 每次对话必读 |
| GL2a 场景规则 | 跨项目合作经验（置信度 0.3-0.7） | `$MEMORY_DATA/AGENTS/GL2-patterns.md` | 按需读取 |
| GL2b 项目规则 | 特定项目积累的经验 | `{项目}/.memory/.rules/` | 进入项目时读取 |
| GL2c 临时约定 | 当前对话观察 | 当前会话（不持久） | 对话结束即过期 |

### GL1 来源

| 来源 | 说明 | 初始置信度 |
|------|------|-----------|
| 从你的 L1 编译 | 从你的价值观/红线直接翻译 | 0.9 |
| GL2 跨项目晋升 | 同一经验在 2+ 项目中被证实有效 | 0.7，你确认后 0.9 |

### GL2 晋升路径

```
GL2b 项目规则（某项目中发现有效）
  │ 同经验在 2+ 项目中出现 → 自动升
  ▼
GL2a 场景规则（跨项目通用）
  │ 累计 5+ 次有效 + 置信度 ≥ 0.7 + 你确认
  ▼
GL1 铁律（永久生效）
```

## 扩展管理

### Skill 架构

Skill 是标准化工作流模块，存放在 `references/`。

| 名称 | 用途 | 加载方式 |
|------|------|---------|
| `memory-assistant` | 日记、规划、进度、周报 | 信号词触发 |
| `monthly-calibration` | 月度蒸馏校准 | 说"月度校准"触发 |
| `reading-assistant` | 读书笔记 | 信号词触发 |

**何时创建新 Skill**：反复描述同样的任务流程时。

**Skill 模板**：
```markdown
# {Skill 名称}
## 触发条件
## 输入
## 执行步骤
## 输出格式
## 质量检查清单
```

### 自定义层

层级架构的扩展，用于领域专精或项目特定内容。

- 何时需要：特定领域的知识/技巧、重要项目的架构/进度、阶段性目标
- 管理：按需创建，定期更新，设 TTL，到期评估是否归档
- 规则：内容已在层级中的不得在自定义层重复存储

### 题目库

- 33 道题目的最新版本在 `07-问卷.md`
- 答案单独存放在各记忆文件（按 I1 路由表）
- 题目有变动时旧答案需重新确认

### 文件版本管理

- 修改前备份
- 修改后更新版本号
- 文件末尾添加变更记录
