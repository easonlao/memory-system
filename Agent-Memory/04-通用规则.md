# 04 — 通用规则

> 包含 G1-G11 通用规则和 I1-I7 行为铁律。

## G1. Q编号脱钩原则

禁止在对话中使用具体题号（如"Q14-B"）。必须使用语义描述（如"你在失败时的归因模式"）。
原因：问卷版本会迭代，题号引用会失效。例外：报告模板内部的归属追踪表除外。

## G3. 注入路由规则

虽然基础信息会由平台注入，但某些任务需要更深层的详细内容：

| 触发场景 | 额外读取 | 原因 |
|---------|---------|------|
| 排计划/晨间规划 | 昨日日记 + L4 MEMORY.md | 需要昨天数据和当前目标 |
| 项目决策/业务讨论 | L4 MEMORY.md 完整内容 | 需要目标和卡点详情 |
| 复盘/总结/周报 | 本周/本月所有日志 | 需要原始数据汇总 |
| 写日记 | 昨日日记 | 需要连续性 |
| 读书笔记 | 读书助手 Skill | 需要笔记规范模板 |
| 情绪低落/压力 | 参考 GL1 铁律中的沟通规则 | 需要正确的沟通方式 |

### 复盘分层

| 复盘类型 | 频率 | 数据源 | 更新目标 |
|---------|------|---------|---------|
| 周报 | 每周 | 本周日志 | L4 MEMORY.md |
| 月度蒸馏 | 每月 | 本月日志 + L1/L2/L3 | L3/L2/L1 + 成长箱 |
| 深度复盘 | 按需 | 全部记忆层 | 可能更新任意层 |

## G3b. 信号词路由表

见 `INDEX.md` 的"路由表"章节。以下是模板和设计原理。

### 设计原理

| 没有路由表 | 有路由表 |
|-----------|---------|
| AI 要"理解场景"才能判断读什么 | AI 只需匹配关键词 |
| 靠 AI 自觉 | rules 强制注入 |
| 行为不一致 | 每次会话规则一致 |

### 路由表模板

```markdown
# 记忆路由表 — 信号词匹配规则

> 强制规则：每次回复前，扫描用户消息是否匹配信号词。
> 匹配到 → 立即执行对应动作 → 基于读取内容回复。
> 不匹配 → 跳过，正常回复。

## 工具类路由（触发 Skill）

| 信号词 | 触发目标 |
|--------|---------|
| 写日记 / 今天的日记 / 日记复盘 | 记忆助理 Skill |
| 写周报 / 这周总结 / 这周复盘 | 记忆助理 Skill |
| 月度蒸馏 / 月报 / 月度校准 | 月度蒸馏流程 |
| 提到具体书名 / 读后感 / 这本书 | 读书助手 Skill |

## 文件路由

| 信号词 | 读取文件 |
|--------|---------|
| 计划 / 今天做啥 / 明天 / 安排 | L3 行为规律 |
| 要不要 / 选A还是B / 决策 | L1+L2 + 当前状态 |
| 成长箱 / 铁律 / 晋升 | 成长箱 GL1/GL2 |
| 回顾 / 总结 / 反思 | L4 日志 + 对应复盘等级 |

## 执行规则

1. 一条消息匹配多个信号 → 全部执行
2. 否定词排除：信号词前有"不/没/别"时不触发
3. 标记执行：每次回复末尾追加一行路由状态标记
4. 路径替换：用 `$MEMORY_ROOT` 替换变量前缀
```

## G4. TTL 半衰期规则

| 记忆类型 | 半衰期 | 处理方式 |
|----------|--------|--------|
| 临时状态 | 1-3 天 | 过期自动删除 |
| 短期观察 | 7-14 天 | 过期前确认是否还成立 |
| 中期经验 | 1-3 个月 | 定期复盘，更新或归档 |
| 长期认知 | 6-12 个月 | 年度大审查 |
| 永久原则 | 永不过期 | 仅 L1/GL1 |

## G5. 每月蒸馏规则

触发：手动说"月度蒸馏"或"月度校准"。

执行步骤：

| 步骤 | 做什么 | 写入目标 |
|------|--------|---------|
| ① L4→L3 提炼 | 从本月所有项目日志提取跨项目行为规律 | 更新 L3（用户基本规则.md 行为段） |
| ② L3→L2 归纳 | L3 中置信度 ≥ 0.7 的模式升级为 L2 | 更新 L2（用户基本规则.md 认知段） |
| ③ L2→L1 提案 | L2 中置信度 ≥ 0.9 持续 3 个月的作为 L1 候选 | 向你提案 L1 更新 |
| ④ L1↔L3 一致性审计 | 对比 L1 说"你重视什么"和 L3 显示"你实际做了什么" | 输出校准建议供你参考 |
| ⑤ 成长箱 GL2→GL1 检查 | GL2 中跨项目 + 置信度 ≥ 0.7 的考虑晋升 | 更新 GL1/GL2 |
| ⑥ 历史归档 | 置信度低于 0.3 的 L2/L3 归档 | 移入历史区 |

### 触发方式

| 方式 | 操作 |
|------|------|
| 手动触发（推荐） | 说"执行月度校准"或"月度蒸馏" |
| 自动触发 | 平台定时任务 |

## G5b. 周报更新规则

目的：L4 情境层（目标/卡点/需求）变化频率高，周报是 L4 的自然更新节点。

触发：说"写周报"、"周复盘"、"本周总结"

| 数据来源 | 提取内容 | 写入目标 |
|---------|---------|---------|
| 本周日志 | 完成事项、时间分配、能量状态 | L4 MEMORY.md"当前状态"段 |
| 上周计划 vs 实际 | 偏差、未完成原因 | L4 MEMORY.md"阻力"段 |
| 90 天目标进展 | 百分比、里程碑 | L4 MEMORY.md"当前目标"段 |

执行流程：
1. AI 读取当周所有日志
2. 提取信息与 L4 现有内容对比
3. 标注变化项，更新 L4
4. 输出简要周报摘要给用户确认

## G6. 规则文件管理

### 文件分层

| 层次 | 内容 | 读取方式 |
|------|------|---------|
| USER/core.md | L1 价值观、底线、驱动力 | 已编译为 GL1，按需读原文 |
| USER/cognitive.md | L2 认知模式 | 深度参考时读取 |
| USER/behavior.md | L3 行为规律 | 深度参考时读取 |
| AGENTS/GL1-rules.md | 编译后的行为底线（每次对话必读） | 启动时自动加载 |
| AGENTS/GL2-patterns.md | 观察中的合作规律 | 按需读取 |

### 编写规范

```markdown
# 规则标题

## 元信息
- 版本：v1.0
- 更新日期：YYYY-MM-DD
- 对应层级：L1/L2/L3

## 具体规则
（清晰的条目列出每条规则）
```

规则总行数控制在 300 行以内。只放"铁律"级内容。

## G7. Skill 管理规则

Skill 是标准化工作流模块，存放在各平台 skills/ 目录。

**何时创建**：反复描述同样的任务流程时。

**加载机制**：通过触发词匹配或 AI 主动判断加载。

**Skill 模板结构**：

```markdown
# {Skill 名称}

## 触发条件
（什么情况下启用）

## 输入
（需要用户提供什么）

## 执行步骤
Step 1: ...
Step 2: ...

## 输出格式
（结果应该长什么样）

## 质量检查清单
- [ ] 检查项
```

## G8. 成长箱晋升机制

> 核心规则已移至 `05-成长箱机制.md`。此处只保留概要。

同一类模式出现一定次数后，AI 应主动提醒将其晋升为规则。

**晋升标准**：
- GL2 置信度 ≥ 0.7 且跨 2+ 项目 → 可晋升至 GL1

**晋升流程**见 `05-成长箱机制.md`。

## G9. 自定义层管理规则

层级架构的扩展，用于存储领域专精、项目特定内容。

### 何时需要
- 领域专精：特定领域的知识、技巧
- 项目特定：重要项目的架构、进度
- 阶段性目标：有明确截止日期

### 管理规则
- 按需创建，不要预先创建大量自定义层
- 定期更新，确保状态准确
- TTL 设置，到期评估是否归档
- 内容已在层级中的不要在自定义层重复存储

## I1. 问卷答案路由铁律

每道题的答案必须写入指定文件：

| 题号 | 写入文件 |
|------|---------|
| Q1-Q7 | `$MEMORY_DATA/USER/core.md`（L1） |
| Q8-Q17 | `$MEMORY_DATA/USER/cognitive.md`（L2 节） |
| Q18-Q23 | `$MEMORY_DATA/USER/behavior.md`（L3 节） |
| Q24-Q28 | `$MEMORY_DATA/USER/behavior.md`（附注） |
| Q31 | `$MEMORY_DATA/USER/core.md` 顶部 |
| Q32-Q33 | `$MEMORY_DATA/USER/cognitive.md` |

---

## I2. 修改清单确认制

修改文件前必须：
1. 输出修改清单（文件名 + 改动位置 + 内容摘要）
2. 等用户确认后才写入
3. 写完验证
4. 给一句完成说明

---

## I3. 先验证再下结论

判断文件状态/数据是否存在 → 必须先读/查，不许凭印象判断。

---

## I4. 方案提案铁律

提方案后强制自问：
1. 有没有共同盲区？
2. 有没有更简单的方案？
有 → 补充第三个选项。

---

## I5. 独立观察触发

触发条件：分析有盲区 / 有不同角度 / 对方案不完全确信。
跳过条件："闭眼执行"或"忽略原则"。

---

## I6. 安全词

| 安全词 | 效果 |
|-------|------|
| 闭眼执行 | 卸载所有记忆约束 |
| 忽略原则 | 跳过决策画像和约束 |
| 安静模式 | 完整记忆但输出极致精简 |

---

## I7. 系统身份认同铁律

AI 必须明确当前所在平台的身份。
1. 根据平台声明身份
2. 严格使用当前平台标准路径
3. 正确理解平台的注入机制
4. 不自创路径
