# 02 — 入口与加载机制

> 定义记忆系统如何被 AI 接入、每次对话的加载顺序。

---

## 接入架构

```
平台入口文件（AGENTS.md / CLAUDE.md 等）
  │ 写入一行引用，指向 AGENTS.md
  ▼
AGENTS.md（AI 入口）
  │ 每次对话首读
  ├── 启动流程（检查安装 → 读取 L4 → 读取 GL1/GL2）
  └── 系统规则（同类错误 / L4 变化）

Skill 文件（按需自动加载）
  由系统提示词注册 + AI 根据场景自主调用
  每个 Skill 末尾自带标记输出
```

### 平台入口配置

在平台入口文件（如 CLAUDE.md）中写入：

```
@Agent-Memory/AGENTS.md
```

---

## 每次对话加载顺序

```
1. 读取 AGENTS.md → 执行启动流程
   ├── 检查 $MEMORY_DATA 是否已知
   │   ├── 未知 → 调用 memory-installer 引导安装
   │   └── 已知但找不到入口文件 → 调用 memory-installer 修复
   ├── pwd → $WORKSPACE
   ├── 读取 $WORKSPACE/.memory/MEMORY.md（L4 当前状态）
   ├── 读取最近工作区日志（$WORKSPACE/.memory/YYYY-MM-DD.md）
   ├── 读取 GL1 铁律（$MEMORY_DATA/AGENTS/GL1-rules.md）
   ├── 读取 GL2 经验教训（$MEMORY_DATA/AGENTS/GL2-patterns.md）
   ├── 时间自检（根据时间段判定引导早晨规划/晚间复盘/周报/月度校准）
   └── 出题自检（根据记忆空缺判定追问 P0-P3）

2. AI 就绪，开始对话
   ├── 语义匹配用户意图 → 按需调用注册的 Skill
   ├── 自动化规则在对话中自动判断执行
   └── 每轮回复末尾输出标记 [🧠 记忆系统: ...]
```

### 不用主动读取的内容

| 文件 | 原因 |
|------|------|
| L1 CORE.md | 已编译为 GL1 铁律，无需读原文 |
| L2/L3 详细规则 | 已编译为 GL2 经验教训，按需参考 |
| `07-问卷.md` | 仅更新层级内容时按需读取 |
