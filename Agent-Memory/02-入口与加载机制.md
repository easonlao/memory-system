# 02 — 入口与加载机制

> 定义记忆系统如何被 AI 接入、每次对话的加载顺序。

---

## 接入架构

```
平台入口文件（AGENTS.md / CLAUDE.md 等）
  │ 写入一行引用，指向入口文件
  ▼
README.md（Agent-Memory入口）
  │ AI 每次对话首读
  ├── 启动流程（固定 5 步）
  └── 系统规则（无 Skill 对应的全局行为）

Skill 文件（按需自动加载）
  由系统提示词注册 + AI 根据场景自主调用
  每个 Skill 末尾自带标记输出
```

### 平台入口配置

在平台入口文件（如 AGENTS.md）中写入：

```
> 记忆系统入口：$MEMORY_ROOT/Agent-Memory/README.md
```

---

## 每次对话加载顺序

```
1. 读取 README.md → 执行启动流程 5 步
   ├── pwd → $WORKSPACE
   ├── 读取 MEMORY.md（当前状态）
   ├── 读取最近工作区日志（脉络衔接）
   ├── 读取 GL1 铁律（行为底线）
   └── 读取 GL2 模式候选（合作偏好）

2. AI 就绪，开始对话
   ├── 语义匹配用户意图 → 按需调用注册的 Skill
   ├── 自动化规则在对话中自动判断执行
   └── 每轮回复末尾输出标记 [🧠 记忆系统: ...]
```

### 不用主动读取的内容

| 文件 | 原因 |
|------|------|
| L1 CORE.md | 已编译为 GL1 铁律，无需读原文 |
| L2/L3 详细规则 | 已编译为 GL2 模式候选，按需参考 |
| `07-问卷.md` | 仅更新层级内容时按需读取 |
