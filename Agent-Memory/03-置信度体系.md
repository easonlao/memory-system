# 03 — 置信度体系

> 定义全系统统一的置信度评分规则，覆盖 L1-L4 和成长箱 GL1/GL2。

---

## 统一置信度标尺

| 置信度 | 含义 | AI 行为 |
|--------|------|---------|
| 0.1 | 单次观察到 | 不记录，未达写入门槛 |
| 0.3 | 重复出现 2-3 次 | 可写入对应层，但不生效（仅记录） |
| 0.5 | 出现频率中等 | AI 可参考，但优先询问你确认 |
| 0.7 | 稳定模式 | AI 自动采用，无需每次确认 |
| 0.9 | 跨场景证实 | 铁律级，修改需你确认 |
| 1.0 | 你主动写入 | 终极来源，无需置信度评估 |

### 核心原则

- **置信度只用于提炼后的模式**（L3/L2/L1/GL2/GL1），不用于原始数据（L4）
- **置信度必须可追溯**——每一条都附有证据（来源、时间、观察次数）
- **置信度只升不降**（除衰减外）——仅在你明确否定时才主动降级

---

## L3 行为规律 — 置信度规则

来源：L4 项目数据

| 置信度 | 条件 |
|--------|------|
| 0.3 | 同一模式在同一项目出现 3-5 天 |
| 0.5 | 同一模式在同一项目持续 2 周 |
| 0.7 | 同一模式跨 2+ 个项目出现 |
| 0.9 | 跨项目 + 持续 3 个月 |

**衰减**：数据不再出现后，置信度每周降 0.1。降到 0.3 以下自动进入历史区。

**晋升触发**：置信度 ≥ 0.7 → 月度蒸馏时考虑升入 L2。

---

## L2 认知模式 — 置信度规则

来源：L3 规律 + 对话观察

| 置信度 | 条件 |
|--------|------|
| 0.3 | 你 1-2 次主动表达 |
| 0.5 | 3-5 次自发表达 + L3 有佐证 |
| 0.7 | 5+ 次且 L3 跨项目佐证一致 |
| 0.9 | 跨 6 个月稳定 + 反例极少 |

**衰减**：6 个月未观察到 → 置信度每月降 0.1。降到 0.3 自动降级到 L3 历史区。

**晋升触发**：置信度 ≥ 0.9 持续 3 个月 → 月度蒸馏时作为 L1 候选提出。

---

## L1 元信息 — 置信度规则

来源：L2 稳定模式 + 你确认

| 置信度 | 条件 |
|--------|------|
| 0.7 | L2 多条一致指向 + AI 首次提案 |
| 0.9 | AI 提案 + 你确认 |
| 1.0 | 你主动写入 |

**L1 特殊规则**：
- 不能自动写入，必须有"你确认"这一步
- 置信度只升不降（你确认过后就是 0.9，不需要再验证）
- 不衰减，永久有效

**退出**：你主动说"这个变了" / L1↔L3 审计连续 3 月提示矛盾。

---

## GL2 经验教训 — 置信度规则

来源：对话 + 项目中的合作规律

| 置信度 | 条件 |
|--------|------|
| 0.3 | 观察到 1 次有效性 |
| 0.5 | 同场景重复 2-3 次有效 |
| 0.7 | 跨场景 5+ 次有效 |
| 0.9 | 跨项目 + 你明确肯定 |

**衰减**：3 个月未触发 → 置信度每月降 0.1。降到 0.3 以下归档。

---

## GL1 铁律 — 置信度规则

| 来源 | 初始置信度 |
|------|-----------|
| 从你的 L1 编译 | 直接 0.9 |
| 从 GL2 晋升（跨项目 + 置信度 ≥ 0.7） | 晋升时 0.7，你确认后 0.9 |

GL1 和 L1 一样：确认后永久锁定，不衰减。

```

GL2  1次有效        2-3次同场景     5+次跨场景       跨项目+你肯定
GL1  —              —              从GL2晋升       你已确认
```
