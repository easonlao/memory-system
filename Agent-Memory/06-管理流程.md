# 06 — 成长箱管理流程

> 定义 AI 侧的自动沉淀流程。这是唯一不由 Skill 封装的工作流——因为它是 AI 在对话间隙自动执行的，不是用户触发的。

---

## 工作流：成长箱沉淀

> AI 自动维护，从对话和工作中沉淀合作规律。

### 子流程 A：对话结束自检（每次对话）

```
1. 本次对话中，有没有发现新的合作规律？
   → 有？置信度多少？→ 写入 GL2c（临时笔记）
   → 没有？跳过

2. 本次对话中，用户有没有明确肯定或否定某种工作方式？
   → 肯定某方式 → GL2c 对应条目的置信度 +0.1
   → 否定某方式 → 对应条目置信度 -0.1

3. 本次对话中，有没有触及已有的 GL1 规则？
   → 触及 → 确认规则是否仍然有效
```

### 子流程 B：成长箱晋升（月度校准后顺便执行）

> 月度校准跑完后，顺带检查成长箱晋升。

```
1. 检查 GL2 置信度
   - 标记跨项目出现的规则
   - 核对置信度数据

2. 晋升评估
   - GL2 置信度 ≥ 0.7 且跨 2+ 项目 → 向你提案晋升 GL1
   - 你确认后写入 GL1

3. 从 L1/L2 编译 GL1
   - L1/L2 本次有更新 → 同步编译到 GL1
   - L1/L2 无变化 → 跳过

4. 归档
   - GL2 置信度降到 0.3 以下 → 归档
```

---

## 置信度规则参考

详细规则见 `03-置信度体系.md`。

| 层级 | 0.3 | 0.5 | 0.7 | 0.9 |
|------|-----|-----|-----|-----|
| L3 | 同一项目 3-5 天 | 同一项目 2 周 | 跨 2+ 项目 | 跨项目 3 个月 |
| L2 | 你 1-2 次表达 | 3-5 次 + L3 佐证 | 5+ 次 + 跨项目佐证 | 6 个月 + 无反例 |
| L1 | — | — | AI 提案阶段 | 你已确认 |
| GL2 | 1 次有效 | 2-3 次同场景 | 5+ 次跨场景 | 跨项目 + 你肯定 |
| GL1 | — | — | 从 GL2 晋升 | 你已确认 |
