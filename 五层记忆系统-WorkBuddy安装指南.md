### 📁 第三步：路径配置（多端协同前提）

> [!important] 🔑 **设计前提：多端协同，不是单机专属**
>
> 本系统的核心设计目标是：**你可能在多个 AI 平台同时使用（WorkBuddy / Cursor / Claude Code 等）**。
> 记忆数据不能锁死在任何一个平台的原生目录里，否则换平台=丢记忆。
>
> **路径变量体系**：所有文件引用使用 `$MEMORY_ROOT` 和 `$WORKSPACE` 变量，AI 执行时替换为实际路径。
> 换机器只需改 `$MEMORY_ROOT = ...` 一处。

**你需要确认两类路径：**

#### 第一类：公共文件夹（★ 多端共享，最重要的部分）

| 目录 | 用途 | 说明 |
|------|------|------|
| `$MEMORY_ROOT/` | 记忆数据主仓库 | 所有平台共用的唯一真实数据源 |
| `$MEMORY_ROOT/核心层/CORE.md` | L1 核心层 | 身份+价值观+底线 |
| `$MEMORY_ROOT/记忆规则/` | L2/L3 规则 | 认知层+行为层详细规则 |
| `$MEMORY_ROOT/rules/` | AI 参考规则 | core-identity.md + behavior-rules.md + 记忆路由表.md |
| `$MEMORY_ROOT/记忆规则/00_动态状态快照.md` | L4 情境层 | 当前目标+瓶颈 |
| `$MEMORY_ROOT/SHADOW.md` | L5 潜意识层 | 言行偏差记录 |
| `$MEMORY_ROOT/技能配置/` | Skill 模板源 | 各平台安装时从此复制 |

> 💡 **关于 `$MEMORY_ROOT`**
>
> 这是公共文件夹根目录的变量占位符。安装时确定实际路径（如 `E:\SynologyDrive\Memory\`），
> 写入 `rules/记忆路由表.md` 的系统常量段。换机器只改这一处。

#### 第二类：WorkBuddy 平台原生路径

| 目录 | 用途 | 说明 |
|------|------|------|
| `~/.workbuddy/IDENTITY.md` + `USER.md` | AI身份+用户画像 | 全局配置，各平台自动读取 |
| `{工作区}/.workbuddy/memory/` | 项目级工作记忆 | MEMORY.md + 每日日志 |
| `~/.workbuddy/skills/` | Skill 工具目录 | 从 `$MEMORY_ROOT/技能配置/` 复制 SKILL.md 过来 |

> 📍 **关键理解**：
> - `$MEMORY_ROOT/` = **数据源头**（你手动维护 / AI 写入）
> - WorkBuddy 通过 `$MEMORY_ROOT` 变量找到记忆文件，按路由表按需读取
> - Skill 不从软链接同步，直接从 `技能配置/` 复制到 `~/.workbuddy/skills/`
> - `IDENTITY.md` / `USER.md` 属于 WorkBuddy 全局配置，与记忆系统解耦

### 📋 第四步：文件检查

系统将检查以下基础文件是否存在：

**公共文件夹（多端共享）：**
- `$MEMORY_ROOT/` 目录是否存在
- `$MEMORY_ROOT/核心层/CORE.md`（核心层，首次问卷后生成）
- `$MEMORY_ROOT/rules/` 目录（AI 参考规则）
- `$MEMORY_ROOT/rules/core-identity.md`（核心身份规则）
- `$MEMORY_ROOT/rules/behavior-rules.md`（行为规则）
- `$MEMORY_ROOT/rules/记忆路由表.md`（信号词路由）
- `$MEMORY_ROOT/SHADOW.md`（潜意识层，可选）

**WorkBuddy 平台原生：**
- `~/.workbuddy/IDENTITY.md`（AI 身份）
- `~/.workbuddy/USER.md`（用户画像）
- `{工作区}/AGENTS.md` 至少存在且含路由表引用
- `{工作区}/.workbuddy/memory/MEMORY.md`（情境层，可选）

> ⚠️ 如果公共文件夹不存在，系统会提示你先创建它——这是多端协同的前提。

### 🎯 第五步：问卷填写

系统将引导您完成33道底层逻辑问卷（问卷完整内容见 `五层记忆系统-框架定义.md`），包括：
1. **核心层（Q1-Q7）** — 您是谁、价值观、决策模式
2. **认知层（Q8-Q17）** — 您怎么思考、如何判断信息
3. **行为层（Q18-Q23）** — 您的行为习惯、工作节奏
4. **情境层（Q24-Q28）** — 您当前的目标、阻力、需求
5. **潜意识层（Q29-Q30）** — 您的知行偏差、对质偏好
6. **协作契约（Q31-Q33）** — 您希望我怎么服务您

### ✅ 第六步：记忆文件生成

完成问卷后，系统将：
- 创建完整的五层记忆文件（写入公共文件夹）
- 生成 `rules/` 下的 AI 参考规则（core-identity.md + behavior-rules.md + 记忆路由表.md）
- 生成专属的《底层洞察与协作契约报告》
- 配置 TTL 管理、Skill 模板、路由表导航等全部功能

### 🧪 第七步：系统验证

系统将进行以下验证：
- 检查所有记忆文件是否创建成功
- 测试规则是否生效（新建对话验证）
- 确认系统就绪状态

## 🧪 系统验证

### 验证步骤

1. **文件完整性检查**：
   - 检查 `~/.workbuddy/IDENTITY.md` 是否存在并有内容
   - 检查 `~/.workbuddy/USER.md` 是否存在并有内容
   - 检查 `~/.workbuddy/rules/` 下是否有 `.mdc` 文件
   - 检查 `.workbuddy/memory/MEMORY.md` 是否存在

2. **功能测试**：
   - **规则生效测试**：新建一个 WorkBuddy 对话，说"用表格对比一下早睡和晚睡的利弊"，观察 AI 是否遵循你定义的交互规则
   - **记忆加载测试**：说"我当前的核心目标是什么"，看 AI 是否能从 MEMORY.md 中回答
   - **Skill 测试**：如果有创建 Skill，说对应的触发词测试是否正常加载

3. **系统就绪确认**：
   - 以上测试全部通过 → 系统就绪 ✅
   - 有问题 → 根据 I3 先验证再下结论的原则排查

### 验证结果模板

| 验证项 | 状态 | 操作 |
|--------|------|------|
| IDENTITY.md | ✅/❌ | |
| USER.md | ✅/❌ | |
| rules/*.mdc | ✅/❌ | |
| MEMORY.md | ✅/❌ | |
| 规则生效 | ✅/❌ | 新建对话测试 |
| Skill 加载 | ✅/❌ | 触发词测试 |

## 📖 使用指南

### 首次初始化（预计 30-45 分钟）

1. **打开 WorkBuddy**：新建一个对话
2. **让 AI 读取框架定义**：说"帮我读取五层记忆系统框架定义并按步骤安装"
3. **完成 Q0 身份确认**（如果没有 IDENTITY.md/USER.md）
4. **完成 33 道问卷**（Q1-Q33），逐一回答
5. **等待 AI 生成记忆文件**：AI 会自动创建所有必要的文件
6. **查看报告**：AI 会生成《底层洞察与协作契约报告》
7. **验证**：新建对话，测试规则是否生效

### 日常使用

1. **晨间规划**：说"今天的计划" → AI 基于 MEMORY.md + 昨日日记帮你规划
2. **写日记**：说"今天的日记" → AI 按 Skill 模板引导你完成日记（含 SHADOW_AUDIT）
3. **周总结**：说"本周总结" → AI 汇总本周日志生成周报
4. **月度蒸馏**：说"月度记忆蒸馏" → AI 清理 MEMORY.md，提取关键洞察
5. **项目决策**：正常对话 → AI 自动遵循你的交互规则和决策画像

### 系统维护

| 频率 | 动作 | 方法 |
|------|------|------|
| 每天 | 写日记 | 说"今天的日记" |
| 每周 | 更新 MEMORY.md 中的情境信息 | 在写周总结时同步更新 |
| 每月 | 记忆蒸馏 | 说"月度记忆蒸馏"或由 automation 触发 |
| 每 3 个月 | 重做 Q8-Q23（认知+行为层） | 说"更新我的认知和行为层信息" |
| 每年 | 重做 Q1-Q7（核心层） | 说"更新我的核心层信息" |
| 随时 | 发现规则需要调整 | 直接告诉 AI "我要修改 XX 规则" |

## 🔄 框架与 WorkBuddy 的对应关系

框架定义文件中的抽象概念在 WorkBuddy 上的实际实现：

| 框架概念 | WorkBuddy 对应 |
|---------|---------------|
| `$MEMORY_ROOT/rules/`（AI 参考规则目录） | `~/.workbuddy/rules/`（自动注入） |
| `IDENTITY.md` + `USER.md` | `~/.workbuddy/IDENTITY.md` + `~/.workbuddy/USER.md` |
| Skill 模板源 `$MEMORY_ROOT/技能配置/` | 复制到 `~/.workbuddy/skills/` |
| 路由表 `$MEMORY_ROOT/rules/记忆路由表.md` | `~/.workbuddy/rules/记忆路由表.mdc` |
| 规则文件后缀 | `.mdc`（WorkBuddy 规则文件规范） |
| 平台入口文件 | `AGENTS.md` 或 WorkBuddy 配置文件 |
| 规则注入机制 | WorkBuddy 自动注入 `rules/*.mdc` 到每次对话 |
