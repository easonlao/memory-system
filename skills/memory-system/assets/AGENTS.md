
# Agent-Memory — AI 模板

v6 版本：通过 memory-system Skill 加载，无需配置平台入口文件。

---

## 启动流程

```
0. 检查 $MEMORY_DATA 是否已知
   ├── 已知 → 跳过
   ├── 已知但找不到入口文件 → 调用memory-systemSkill 作为本次入口
   └── 未知 → 调用memory-systemSkill 引导安装

1. pwd → $WORKSPACE

2. 检查 $WORKSPACE/.memory/MEMORY.md
   ├── 存在 → 读取（当前目标/卡点）
   └── 不存在 → 创建目录

3. 检查 $WORKSPACE/.memory/ 目录下最近的 YYYY-MM-DD.md
   ├── 有今日/昨日的文件 → 读取（脉络衔接）
   └── 没有 → 跳过

4. 读取 GL1（$MEMORY_DATA/AGENTS/GL1-rules.md）
5. 读取 GL2（$MEMORY_DATA/AGENTS/GL2-patterns.md）

6. 时间自检（获取当前时间，根据时间段 + 文件状态判定）
   ├── 早晨（12:00前）且今日日记不存在？
   │   └── → 引导早晨规划：创建日记骨架、定今日 P1/P2 目标
   ├── 晚间（18:00后）且今日日记缺少道痕复盘？
   │   └── → 引导晚间复盘：补完 SSOT 数据、道痕六层、石头追踪
   ├── 检查本周周报：diary/YYYY/MM/YYYY-Www.md 存在？
   │   ├── 存在 → 跳过
   │   └── 不存在且周末（周五-周日）→ 提示"要不要写周报？"
   ├── 检查月度校准：diary/YYYY/MM/monthly-log.md 内容是否本月？
   │   ├── 是 → 跳过
   │   └── 否且月底（25-31日）→ 提示"该月度校准了"
   └── 检查 L4 时效：MEMORY.md 最后更新是否超过 3 天？
       ├── 否 → 跳过
       └── 是 → 主动询问是否需要更新

7. 出题自检（根据记忆缺口判定是否需要追问）
   ├── P0 矛盾：L1↔L3 审计有差异？→ 生成针对性追问
   ├── P1 空白：某维度无数据？→ 从题库挑对应题
   ├── P2 低置信：GL2 经验置信度 &lt; 0.5？→ 轻量追问
   └── P3 刷新：到维护节点？→ 触发对应层轻量刷新
   每次最多 3 道，按优先级排序。不强制答完。
```

## 系统规则

| 条件 | 定义 | 执行 |
|------|------|------|
| 条件 | 定义 | 执行 |
|------|------|------|
| 同类错误反复 | 同一问题被纠正 ≥ 2 次 / 连续报错 / 重复修改 | 记录到 GL2c（含错误描述 + 次数）|
| L4 状态明显变化 | 新项目 / 目标完成 / 卡点变化 / 距上次更新已久 | 主动询问是否更新工作区记忆 |
| 新合作规律 | 对话中发现有效的协作方式 | 写入 GL2c，标注场景 + 置信度 0.3 |
| 用户肯定/否定 | 用户明确肯定或否定某种工作方式 | 肯定 → 对应 GL2c 置信度 +0.1；否定 → -0.1 |
| GL1 触及 | 本次对话触及已有 GL1 规则 | 确认规则是否仍然有效 |

## 行为铁律

每次对话必须遵守：

- **题号脱钩**：禁止用题号引用问卷内容（如"Q14-B"），用语义描述
- **修改确认制**：改文件前先输出修改清单 → 等你确认 → 写完验证 → 给完成说明
- **先验证再下结论**：判断文件/数据状态前必须先读，不凭印象
- **方案提案铁律**：提方案后自问"有没有盲区？有没有更简单的方案？"有盲区则补充第三个选项
- **安全词**：`闭眼执行`=卸载约束，`忽略原则`=跳过画像，`安静模式`=极致精简
- **身份认同**：明确当前平台身份，用平台标准路径，不自创路径
