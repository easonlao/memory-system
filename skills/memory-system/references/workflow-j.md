# 工作流 J：帮助系统（MS-HELP）

触发："MS-HELP" / "帮助" / 用户用自然语言表达模糊需求时自动调用

目的：统一入口，用户只需说"MS-HELP 我想做X"，AI 判断路由到正确工作流。

## J1：理解用户意图

1. 检查用户输入：
   - 空（只说 "MS-HELP"）→ 输出系统概览 + 推荐下一步
   - 带具体描述（"MS-HELP 我想记录今天的工作"）→ 匹配最佳工作流
   - 带问题（"MS-HELP 道痕是什么"）→ 解释概念
   - 带故障描述（"MS-HELP 记忆系统没反应"）→ 诊断排查

2. 用户状态感知：
   - 未安装 → 安装指南
   - 新用户（安装 < 7 天或日记 < 5 篇）→ 推荐最小可用节奏
   - 活跃用户（有持续日记）→ 根据数据缺口推荐

## J2：匹配工作流

根据用户自然语言描述，自动匹配最佳工作流，AI 自动读取对应文件：

## J3：推荐下一步

根据用户当前数据状态输出推荐：

| 数据状态 | 推荐内容 |
|---------|----------|
| 今日未写日记 | "今天还没有记录，要不要写日记？" |
| 本周未写周报且已是周末 | "本周还没写周报，要不要整理一下？" |
| 本月未校准且已是下旬 | "本月还没做月度校准，要不要跑一次？" |
| L4 超过 3 天未更新 | "当前工作区记忆超过 3 天没更新了" |
| 一切正常 | "一切正常。有需要随时找我。" |

末尾追加标记：`[🧠 Agent-Memory: MS-HELP]` + `[🧠 路由: 系统]`

执行完毕后，按 SKILL.md 通用推荐规则推荐下一步。
