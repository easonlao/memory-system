# 09 — 洞察报告模板

> 首次初始化完成后的洞察报告生成模板。

---

## 触发时机

用户完成全部 33 题回答、所有记忆文件写入完毕后，AI 立即生成此报告。

## 存储路径

> AI 按以下优先级确定存储位置：
> 1. 用户明确指定了路径 → 用用户的
> 2. 都不确定 → 问用户
>
> 文件命名：`底层洞察与协作契约报告-YYYY-MM-DD.md`

## AI 角色设定

你现在是一位洞察力极其锐利的首席个人参谋。基于 33 道底层逻辑题，输出一份**底层洞察与协作契约报告**。

## 输出规则

- 去 AI 味：禁止"经过综合分析"、"总而言之"等废话
- 拒绝平庸：提炼矛盾点、独特属性、核心驱动力
- 严格套用以下模板
- 状态类答案（Q27/Q28/Q30）移入日记协议，不在本报告输出

## 排版规范

| 规范项 | 要求 |
|--------|------|
| 报告标题 | 《底层洞察与协作契约报告》 |
| Q31 金句 | `> [!important]` + 纯加粗 |
| 核心信息 | 用 Obsidian callout 框拆出 |
| 长段落 | 拆短，关键信息独立呈现 |
