# 工作流 E：晚间复盘

触发："今日复盘" / "晚间总结" / "写日记"

## E1：SSOT 数据补完
补全日志中以下数据（引导用户提供）：
- 入睡/起床时间
- 睡眠质量
- 总番茄数
- 能量状态
- 情绪评分（1-10）

## E2：道痕引导
逐层提问，**每次只问一层**，用户回答后再进行下一层。记录用户原文到对应层。

**Q1：事实**
> 今天最让你起波澜的一件事是什么？只讲发生了什么，不评价不抒情。

**Q2：第一反应**
> 当时你的第一反应是什么？不是正确答案，是最直接的感觉。

**Q3：贪婪**
> 顺着那股劲，你其实想得到什么？不要用大词，写具体的。

**Q4：恐惧**
> 你其实在害怕什么？写最直接的那个怕。

**Q5：自洽**
> 你给了自己什么理由来消化这件事？你是怎么说服自己"这样也挺好"的？

**Q6：主石头**
> 今天捞出来的主石头是哪块？一句话：贪婪还是恐惧？具体是什么？

**Q7：人选练习**
> 如果明天再遇到同样的事，你准备怎么选？

## E2.5：冲突扫描（G14 晚间复查）
参考 AGENTS.md 的 G14 第三层协议，扫描今日所有对话中的冲突记录。

## E3：石头追踪
完成道痕后，自动执行：
1. 记录今日主石头到日记 frontmatter `stones` 字段
2. 与近 7 天日记中的 stones 做对比，输出一句观察："这块'{石头}'本周出现第 N 次"
   - 不做自动写入，不做自动晋升
   - 观察结果仅作为对话反馈

## E4：GL2b → GL2a 晋升检查
读取各项目的 `.rules/` 文件，检查是否有置信度 ≥ 0.7 **且**跨项目出现的规则：
- 满足条件 → 输出提案并询问用户："规则'{描述}'在 2+ 项目中出现（置信度 {N}），是否晋升到全局规则？"
  - 用户确认 → 写入 `$MEMORY_DATA/AGENTS/GL2-patterns.md`，格式：
    ```
    - {描述} | 置信度: {N} | 来源: {项目A}, {项目B} | 确认于: {YYYY-MM-DD}
    ```
  - 用户拒绝 → 跳过
- 不满足条件 → 跳过

## E5：写入日记
按 SKILL.md 日记模板写入 `$MEMORY_DATA/USER/diary/YYYY/MM/YYYY-MM-DD.md`

## E6：L4 & 项目总览同步
- 如果今天有工作产出变化 → 更新 `$WORKSPACE/memory/MEMORY.md`
- 如果项目状态变化 → 更新 `$MEMORY_DATA/AGENTS/project-index.md`
- **清理**：检查 project-index.md 中的条目对应的 `$WORKSPACE` 目录是否仍然存在，不存在的条目询问用户是否删除

## E7：明日预告
- 如果今天是周日 → "今天是周日，要不要顺便写周报？"

末尾追加标记：`[🧠 Agent-Memory: 晚间复盘]` + `[🧠 路由: L1]`

执行完毕后，按 SKILL.md 通用推荐规则推荐下一步。
