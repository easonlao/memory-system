# 月度校准

> 你说"月度校准"时执行。从本月日记中提炼行为模式，逐层升级记忆层级。
> 逐步骤执行并输出中间结果，让你看到数据到结论的推导过程。

---

## 输入

本月所有日记

## 步骤 ①：日记 → L3 提炼

> 从本月日记中提取行为规律

```
输入：本月所有日记
处理：扫描出现 3+ 次以上的行为模式
输出：L3 候选列表（含置信度和证据来源）
写入：$MEMORY_ROOT/记忆规则/用户基本规则.md（行为段）

例：
"本月 3 个项目都在截止日前 2 天集中冲刺"
→ L3 候选 "截止日前冲刺" 置信度 0.5
```

## 步骤 ②：L3 → L2 归纳

> L3 中置信度 ≥ 0.7 的模式升级为 L2 认知模式

```
输入：L3 当前列表（关注置信度 ≥ 0.7 的条目）
处理：
  - 检查该模式是否已存在 L2
  - 存在 → 合并证据，提升置信度
  - 不存在 → 创建 L2 新条目
输出：更新的 L2 列表
写入：$MEMORY_ROOT/记忆规则/用户基本规则.md（认知段）
```

## 步骤 ③：L2 → L1 提案

> L2 中置信度 ≥ 0.9 持续 3 个月的作为 L1 候选

```
输入：L2 当前列表（关注置信度 ≥ 0.9 的条目）
处理：
  - 该模式已稳定持续 3 个月以上？
  - 跨场景一致？（工作/生活/多个项目）
  - 反例极少？
输出：L1 候选提案
动作：向你输出提案，等你确认后才写入 L1
```

## 步骤 ④：L1 ↔ L3 一致性审计

> 对比 L1 说"你重视什么"和日记显示"你实际做了什么"

```
处理：
  - 逐条比对 L1 值与本月日记数据
  - 计算差异
  - 标记持续矛盾的条目

输出格式：
📊 L1↔L3 一致性审计
━━━━━━━━━━━━━━━━
 ✅ L1「结果重于过程」↔ 本月完成率 85% → 一致
 ⚠️ L1「健康优先」↔ 本月平均睡眠 5.2h → 差异显著
 ❌ L1「不将就」↔ 本月勉强接受了 2 次合作 → 建议讨论
```

## 步骤 ⑤：更新 GL1 编译

> L1/L2 更新后，重新编译 GL1 铁律。
> 编译源：本次更新后的 L1 + L2。

```
动作：
1. 检查本次 L1/L2 是否有实际变更
   ├── 有变更 → 同步编译到 GL1（$MEMORY_ROOT/成长箱/01-铁律.md）
   └── 无变更 → 跳过

2. 编译规则：L1 底线 → GL1 铁律，L2 交互偏好 → GL1 沟通规则
```

## 步骤 ⑥：成长箱晋升检查

> 月度校准后执行成长箱晋升评估。
> 参考 `06-管理流程.md` 的晋升流程。

```
输入：GL2 当前列表（$MEMORY_ROOT/成长箱/02-模式候选.md）
处理：
  - 标记跨项目出现的规则
  - 置信度 ≥ 0.7 且跨 2+ 项目 → 向你提案晋升 GL1
  - 置信度降到 0.3 以下 → 归档
输出：晋升提案 / 归档列表
```

## 输出汇总

```
📋 本月校准结果
━━━━━━━━━━━━━━━━
L3 提炼：3 条新行为规律
L2 归纳：2 条升级为认知模式
L1 提案：1 条待确认
L1↔L3 审计：1 处差异建议关注
→ 步骤⑤ GL1 编译：{已更新/无需变更}
→ 步骤⑥ 成长箱：{X 条晋升提案 / X 条归档}
```
