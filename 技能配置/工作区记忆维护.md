# 工作区记忆维护

> **全局准则**：
> 1. 格式严格，不随意增减字段
> 2. 工作区日记是原稿，MEMORY.md 是提炼，项目总览是索引
> 3. 无数据就留空，不编造

**常量**：`$MEMORY_ROOT` = `E:\SynologyDrive\Memory\源文件`

---

## 工作流 A：写工作区日记（每次 session 结束）

> 每次对话结束时，将此 session 的内容概括写入当天工作区日记。
> 一天可能有多个 session，追加到同一个文件。

### 路径

`$WORKSPACE/.memory/daily/YYYY-MM-DD.md`

### 模板

```markdown
# 工作日志 - YYYY-MM-DD
工作区: {$WORKSPACE}

## Session {序号} {HH:MM}
- 主题：{主要话题}
- 关键产出：{代码/方案/文档/决策}
- 待办：{未完成需要下次继续的事项}
```

### 执行步骤

```
1. daily/ 目录是否存在？
   ├── 存在 → 继续
   └── 不存在 → 创建

2. 今日日记已存在？
   ├── 存在 → 读取，追加新 session
   └── 不存在 → 创建新文件

3. 追加完毕后，检查是否需要更新 MEMORY.md
   └── 本次 session 有状态变化？→ 触发工作流 B
```

---

## 工作流 B：更新 L4 MEMORY.md

> 从工作区日记中提炼核心状态。只记当前快照，不记历史细节。
> 触发：工作流 A 完成后自动判断，或自动化路由触发。

### 步骤 1：确定工作区类型

- 当前目录有 `package.json` / `.git` / 典型项目结构 → **项目**
- 其他 → **日常**

### 步骤 2：从日记中提炼关键信息

读取当日及近几日工作区日记，提取：

- **进展**：完成了什么
- **卡点**：卡在什么地方
- **决策**：确定了什么方案
- **当前目标**：正在做什么

### 步骤 3：按模板写入

**项目模式**：

```markdown
# 工作区记忆

类型: 项目
路径: {$WORKSPACE}

## 当前状态
- 项目：{项目名}
- 目标：{当前核心目标}
- 进展：{最近完成的关键事项}
- 卡点：{当前卡住的问题}

## 关键记录
- 决策：{已确定的方案}
- 上下文：{分支/配置/环境信息}

---
最后更新: {YYYY-MM-DD}
```

**日常模式**：

```markdown
# 工作区记忆

类型: 日常
路径: {$WORKSPACE}

## 当前状态
- 最近话题：{这段时间在聊什么}
- 关键产出：{产生的可用产物}

---
最后更新: {YYYY-MM-DD}
```

### 步骤 4：写入文件

路径：`$WORKSPACE/.memory/MEMORY.md`
- 已存在 → 读取旧内容，只更新变化的字段
- 不存在 → 创建后写入

---

## 工作流 C：同步到项目总览

> 每次更新 MEMORY.md 后自动执行。

### 路径

`$MEMORY_ROOT/项目总览.md`

### 格式

```markdown
# 项目总览

| 工作区路径 | 类型 | 描述 | 最后更新 |
|---|---|---|---|
| path/to/proj | 项目 | 完成XX模块，当前做API设计 | 2026-05-10 |
| path/to/chat | 日常 | 讨论了YY话题 | 2026-05-08 |
```

### 执行步骤

```
1. 项目总览.md 已存在？
   ├── 存在 → 读取
   └── 不存在 → 创建，写入表头

2. 当前工作区路径已有对应行？
   ├── 有 → 更新描述 + 最后更新
   └── 没有 → 追加新行

3. 写入项目总览.md
```

**描述生成规则**：从 MEMORY.md 当前状态拼接成一句话。
- 项目：`{项目名}，进展：{进展}`（有卡点则追加 `，卡点：{卡点}`）
- 日常：`{最近话题}`
