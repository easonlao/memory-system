# 🧠 唤醒记忆系统 — 记忆自检 Skill

## 功能

不是"加载器"——各平台的系统配置已自动注入基础记忆。
本 Skill 做一件事：读文件 → 判断状态 → 输出标准报告 → 确认可用后直接干活。

## 触发命令

| 命令 | 场景 |
|------|------|
| `检查记忆` / `加载记忆` / `记忆状态` | 正式启动 |
| `启动记忆系统` / `唤醒记忆` | 明确指令 |
| `我美吗` | 彩蛋命令 |
| 新任务开始时主动执行 | 最佳实践 |

## 自检流程（4步）

### 步骤 1：读取 INDEX.md

读取 `$MEMORY_ROOT/五层记忆系统/INDEX.md`，了解系统结构和路由表。

### 步骤 2：检查核心文件

| 文件 | 检查项 |
|------|--------|
| L4 MEMORY.md | 是否存在、行数、是否超 150 行 |
| 今日日志 `YYYY-MM-DD.md` | 今天是否存在 |
| L1 CORE.md | 是否可读 |
| L2/L3 用户基本规则.md | 是否可读 |
| GL1 铁律 | 是否可读 |

### 步骤 3：输出标准报告

```
🧠 记忆系统自检报告
━━━━━━━━━━━━━━━━━━━
📍 平台：[当前平台]
⏰ 时间：[YYYY-MM-DD HH:MM]

📄 L4 情境层
   └─ ✅ 存在，X 行 [正常]
   └─ ⚠️ 空文件（无当前状态）
   └─ 🔴 不存在

📄 今日日志
   └─ ✅ 已存在
   └─ ⚠️ 尚无

📋 文件状态
   L1 CORE.md    ── ✅/⚠️/❌
   L2/L3 规则   ── ✅/⚠️/❌
   L4 情境层    ── 见上方
   GL1 铁律     ── ✅/⚠️/❌

🔍 结论
   └─ 🟢 正常，可直接工作
   └─ 🟡 部分缺失（列出）
   └─ 🔴 需初始化
```

### 步骤 4：根据结论行动

| 结论 | 行动 |
|------|------|
| 🟢 正常 | 静默完成，直接进入用户任务 |
| 🟡 部分缺失 | 标注缺失项，不阻塞任务 |
| 🔴 需初始化 | 提示用户，等确认后再操作 |

## 禁止行为

1. ❌ 不要因为 MEMORY.md 为空就停下来问"是否启用记忆系统"
2. ❌ 不要输出冗长报告后再问"要我初始化吗"
3. ❌ 不要阻塞用户任务
4. ❌ 不要编造数据
