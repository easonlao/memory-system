# 🧠 唤醒记忆系统 — 安装与健康入口

> `$MEMORY_DATA` 未设置时自动调用。负责安装（目录 + 问卷填充 + 报告）和健康检查。

---

## 工作流 A：首次安装

### 1. 设置目录

```
1. 询问用户选择 $MEMORY_DATA 路径
   mkdir -p $MEMORY_DATA/{USER,AGENTS}

2. 告知用户：$MEMORY_DATA 是唯一需要备份的目录
```

### 2. 问卷填充

引导用户逐部分回答 `$MEMORY_ROOT/五层记忆系统/07-问卷.md`。

**第一部分 核心层（Q1-Q7）→ USER/L1-核心.md**
**第二部分 认知层（Q8-Q17）→ USER/L2-L3-规则.md（L2 节）**
**第三部分 行为层（Q18-Q23）→ USER/L2-L3-规则.md（L3 节）**
**第四部分 情境层（Q24-Q28）→ 附注记录**
**协作契约（Q31-Q33）→ L1-核心.md 顶部**

每答一题立即写入对应文件。Q29-Q30 保留辅助参考。

### 3. 编译 GL1

```
读取 USER/L1-核心.md → 编译初始规则 → 写入 AGENTS/GL1-铁律.md
```

### 4. 生成报告

按 `09-报告模板.md` 生成 → 输出到 `$MEMORY_DATA/底层洞察与协作契约报告.md`

### 5. 完成

输出安装摘要，告知备份事项。

---

## 工作流 B：健康检查

触发：启动时 `$MEMORY_DATA` 已知，或用户说"检查记忆"

```
1. 扫描结构 → 列出各文件状态（有内容/空/缺失）
2. 🟢 全部就绪 → 静默完成
   🟡 部分缺失 → 列出缺失项，询问是否修复
   🔴 无 $MEMORY_DATA → 引导安装
```

---

## 工作流 C：结构修复

触发：健康检查发现缺失，用户确认修复

```
1. 缺失文件 → 创建空模板
2. 内容为空 → 引导补充问卷对应部分
3. 完成后重新检查
```

---

## 工作流 D：迁移恢复

触发：用户说"迁移记忆"、"更换设备"

```
1. 告知：复制旧设备 $MEMORY_DATA 到新设备即可
2. 设定 $MEMORY_DATA 指向新路径
3. 运行健康检查确认完整性
```
