# 🧠 唤醒记忆系统 — 安装与健康入口

> 这是整个记忆系统的入口 Skill，负责三件事：安装、健康检查、恢复。
> 启动流程中 $MEMORY_DATA 未设置时自动调用。

---

## 工作流 A：首次安装（$MEMORY_DATA 不存在）

触发：无 `$MEMORY_DATA`，需引导用户创建

```
1. 询问用户：运行时记忆库放在哪里？
   - 建议路径：~/.memory-data/ 或 用户指定
   - 确认路径后 设为 $MEMORY_DATA

2. 创建目录结构
   mkdir -p $MEMORY_DATA/{USER,AGENTS}

3. 创建初始文件
   - USER/L1-核心.md（空模板）
   - USER/L2-L3-规则.md（空模板）
   - AGENTS/GL1-铁律.md（空模板）
   - AGENTS/GL2-模式候选.md（空模板）
   - AGENTS/项目总览.md（空表头）

4. 告知用户：$MEMORY_DATA 已就绪
   - 这是唯一需要备份的目录
   - 迁移时只需复制 $MEMORY_DATA 到新设备
```

## 工作流 B：健康检查（$MEMORY_DATA 已存在）

触发：启动时 `$MEMORY_DATA` 已知，或在对话中说"检查记忆"、"记忆状态"

```
1. 扫描 $MEMORY_DATA 结构
├── 检查 USER/L1-核心.md       → 存在/缺失
├── 检查 USER/L2-L3-规则.md    → 存在/缺失
├── 检查 AGENTS/GL1-铁律.md    → 存在/缺失
├── 检查 AGENTS/GL2-模式候选.md → 存在/缺失
└── 检查 AGENTS/项目总览.md    → 存在/缺失

2. 扫描 $MEMORY_ROOT 结构（框架完整性）
├── 检查 五层记忆系统/README.md  → 存在/缺失
├── 检查 技能配置/ 主要 Skill    → 存在/缺失
└── 报告缺失项

3. 输出报告
🟢 状态良好 → 静默完成
🟡 部分缺失 → 列出缺失项，询问是否修复
🔴 严重缺失 → 提示重新安装
```

## 工作流 C：结构修复

触发：健康检查发现结构问题，用户确认修复

```
1. 缺失 USER/ 或 AGENTS/ 文件 → 创建空模板文件
2. 缺失 框架文件 → 告知用户重新下载框架
3. 文件存在但内容异常 → 备份旧文件后重建
4. 完成后重新运行健康检查
```

## 工作流 D：迁移恢复

触发：用户说"迁移记忆"、"恢复记忆"、"更换设备"

```
1. 询问用户新的 $MEMORY_DATA 路径
2. 告知：只需将旧设备的 $MEMORY_DATA 复制到新设备
3. 设置 $MEMORY_DATA 指向新路径
4. 运行健康检查确认完整性
```
