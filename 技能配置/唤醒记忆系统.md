# 🧠 唤醒记忆系统 — 安装与健康入口

> 整个记忆系统的入口 Skill。负责安装（创建结构 + 问卷填充）、健康检查。
> `$MEMORY_DATA` 未设置时自动调用。

---

## 工作流 A：首次安装（$MEMORY_DATA 不存在 + 问卷填充）

### 阶段 1：设置目录

```
1. 询问用户：运行时记忆库放在哪里？
   - 建议：~/.memory-data/ 或自定义路径
   - 确认后设为 $MEMORY_DATA

2. 创建目录结构
   mkdir -p $MEMORY_DATA/{USER,AGENTS}
```

### 阶段 2：问卷填充

引导用户逐部分回答以下问卷（题目见文末附录）。

**第一部分：核心层（Q1-Q7）→ 写入 L1-核心.md**
```
每答完一题，立即写入对应字段：
- Q1/Q3/Q5 → 核心驱动力
- Q2 → 佩服的品质
- Q4 → 自我描述
- Q6 → 认知盲点
- Q7 → 愿景
```

**第二部分：认知层（Q8-Q17）→ 写入 L2-L3-规则.md（L2 节）**
```
- Q8-Q12 → 决策习惯/思维方式
- Q13-Q17 → AI 交互偏好/沟通规则
```

**第三部分：行为层（Q18-Q23）→ 写入 L2-L3-规则.md（L3 节）**
```
- Q18-Q22 → 能量周期/工作节奏/习惯
- Q23 → 成就感来源（简答）
```

**第四部分：情境层（Q24-Q28）→ 写入 L2-L3-规则.md 附注**
```
当前目标和卡点。仅做初始记录，后续由工作区日志迭代。
```

**第五部分：协作契约（Q31-Q33）→ 写入 L1-核心.md 顶部**
```
- Q31 → 金句（置顶）
- Q32-Q33 → 补充说明
```

### 阶段 3：编译初始 GL1

```
读取 USER/L1-核心.md 的底线/价值观
编译为初始 GL1 规则，写入 AGENTS/GL1-铁律.md
```

### 阶段 4：生成洞察报告

```
读取所有问卷答案，按文末的报告模板生成《底层洞察与协作契约报告》
输出到 $MEMORY_DATA/底层洞察与协作契约报告.md
```

### 阶段 5：完成

```
1. 输出简要的安装摘要（各层填充情况）
2. 告知：
   - $MEMORY_DATA 是唯一需要备份的目录
   - 迁移时复制整个 $MEMORY_DATA 到新设备即可
   - 日常使用不需再碰这些文件，AI 会自动维护
```

---

## 工作流 B：健康检查（$MEMORY_DATA 已存在）

触发：启动时 `$MEMORY_DATA` 已知，或用户说"检查记忆"、"记忆状态"

```
1. 扫描 $MEMORY_DATA 结构
├── USER/L1-核心.md           → 有内容/空/缺失
├── USER/L2-L3-规则.md        → 有内容/空/缺失
├── AGENTS/GL1-铁律.md        → 有内容/空/缺失
├── AGENTS/GL2-模式候选.md    → 有内容/空/缺失
└── AGENTS/项目总览.md        → 有内容/空/缺失

2. 输出报告
🟢 全部就绪 → 静默完成
🟡 部分缺失 → 列出缺失项，询问是否修复
🔴 无 $MEMORY_DATA → 引导安装
```

---

## 工作流 C：结构修复

触发：健康检查发现缺失，用户确认修复

```
1. 缺失文件 → 创建空模板
2. 内容为空 → 引导补充问卷对应部分
3. 完成后重新健康检查
```

---

## 工作流 D：迁移恢复

触发：用户说"迁移记忆"、"更换设备"

```
1. 告知：只需将旧设备的 $MEMORY_DATA 目录完整复制到新设备
2. 设定 $MEMORY_DATA 指向新路径
3. 运行健康检查确认完整性
```
